Introduction¶

In this report, the comparison of Scattering Wavelet Transforms with Deep Neural Networks will be addresses. We will show the concept of symmetry and the importance of invariance in machine learning models, especially for signal processing applications (in this case handwritten digit recognition). The report compares the use of black box convolutional neural networks (in recent years, these models have been incredibly expressive in solving these problems) and the use of models caused by signal processing. The type of these problems is very important in signal processing: scattering wavelet transform.

In deep learning, a convolutional neural network (CNN, or ConvNet) is a class of deep neural networks, most commonly applied to analyzing visual imagery. They are also known as shift invariant or space invariant artificial neural networks (SIANN), based on their shared-weights architecture and translation invariance characteristics. They have applications in image and video recognition, recommender systems, image classification, medical image analysis, natural language processing, and financial time series.

The output of the convolution is usually fed to a so-called "detector" layer, which consists of a non-linear activation function, usually ReLU in most implementations. Where the kernel is inconsistent with the input, ReLU "turns off" the input. Therefore, the output tensor is much sparse. Then perform a downsampling operation on this output, which is called "merging". The pooling layer essentially down-samples the neighborhood by summarizing the output in the neighborhood into representative statistical data. It is said that this makes the network's transformation of the input flexible, depending on the size of the neighborhood under consideration. Later we will see that the wavelet scattering transform also uses pooling as one of the operations of translation invariance.

Wavelet scattering transform follow the spirit of decomposing the input function into a set of special basis functions that are invariant. Fourier transform attempts to decompose a function into Fourier basis, that is, to put it simply, try to express the function as a linear combination of Fourier basis functions, which is a complex sinusoid. Wavelet transform attempts to perform the same operation and decompose the function into wavelet basis. Both methods introduce some deformation invariance. Wavelet scattering attempts to decompose in a nonlinear way-as far as I know, it uses complex modulus functions as nonlinearities.

The rest of this report consists of following parts. Firstly, I will used VGG16 and ResNet18 pretrained on ImageNet as a fixed feature extractor for MNIST and a scattering wavelet transform on MNIST as another method of extracting deformation invariant features. In addition, I used Logistic regression and random forest for classification based on the extracted features. Finally, I will compare the results and provide high level explanation of the performance difference.

Methodology and Implementation¶

General description of the CNN is as follows:¶

  • Use pretrained Alexnet, VGG16 and ResNet18 models from PyTorch

  • Create modified versions of Alexnet, VGG16 and Resnet18 after removing the last fully connected layer from the architecture and use the output of the penultimate layer as extracted features. The initialization of the class constructs the architecture ommitting the last layer, if required. Forward function is updated to not pass the input through the last layer.

  • Extract and save the MNIST dataset. According to the description, the default resolution of MNIST images are 28 x 28 pixels and the input sizes of ImageNet dataset is 3 x 224 x 224 where 3 indicates the RGB channel. Therefore, MNIST is transformed by scaling to 3 x 224 x 224.

  • Push the transformed MNIST dataset through the networks with the shape of the features extracted from the models are as follows: AlexNet and VGG16 (60000, 4096), ResNet18 (60000, 512).

  • Train and store the trained models based on extracted data. As stated before, we train Logistic Regression and Random Forests for this report. The code supports Gradient Boosting Trees also and can also easily be extended to SVMs. The code also supports choosing parameters using cross validation, however that was ommitted due to lack of enough compute resources.

Wavelet Scattering Transform:¶

  • The code uses wavelet scattering implementation from Kymatio which implements the wavelet scattering transform as a convolutional neural network. The documentation is hosted here: kymat.io

  • The shape of the features from the wavelet scattering transform is (60000, 81, 7, 7) which is flattend to (60000, 3969)

Models trained using the features above¶

Logistic Regression and Random Forests were trained as the pick for 1 linear and 1 non-linear model to use. Of course, as stated earlier, one could be more rigorous and include more such models to train and run cross validation to choose the best params, but the hope is that the gist of the analysis is captured by these two.

The models are already trained and stored on disk and is defined in utils.py in the function train_classifier. Similarly, in order to test the classifier, i.e. get predictions on one of the saved test datasets, we only need to change to test_classifier

In order to train logistic regression on features extracted from AlexNet, one could call the function as follows:

from utils import train_classifier
train_classifier("log_reg", dataset="alexnet", cv=False, save_to_disk=True)

cv=False means that we do not want to run Cross Validation. save_to_disk=True is self explanatory.

Conclusion¶

In conclusion, we have perform several tests to address and compare Logistic Regression and Random Forest. However, these test are only performed on 1 training and test data pair, which is also very important. Generally, it is better to mix a given test data set with a training data set, and then randomly divide it into training and test data for a specific experiment. Running multiple such experiments and obtaining a dispersion of accuracy values is important to consider the dependence of generalization accuracy on a particular training/test set. The usual practice is to draw a box plot of the accuracy values of multiple experiments for each model-data set pair, and then compare the medians of these values. What I have done in this report is just a fairly soft version of what I have described. Deformed images can also be added to the data set, and experiments can also be performed on this.

It is also important to note that although the wavelet scattering transform provides some theoretical guarantee for the invariance of deformation, deep CNN does not provide any such guarantee. CNN has been designed and fine-tuned for each application to suit that specific application. In this case, the pre-trained CNN used will be pre-trained on the ImageNet dataset that is not completely "similar" to MNIST. This may also be the reason for the lagging performance of models trained with those CNN functions.
